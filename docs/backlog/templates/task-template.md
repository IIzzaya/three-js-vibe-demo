# 任务模板

以下是标准任务格式，复制到 `docs/backlog/active/sprint-{slug}.md` 中使用：

```markdown
## [T{n}] 任务标题

- **状态**: 🔵 待开发
- **类型**: 🆕 功能 | 🐛 修复 | ♻️ 重构 | 🎨 美术 | 📝 设计 | ⚡ 优化
- **点数**: N
- **优先级**: P0（必须）/ P1（重要）/ P2（一般）
- **相关文档**: docs/game-design/xxx.md
- **验收标准**:
  - [ ] 标准 1
  - [ ] 标准 2
  - [ ] 标准 3
- **DebugBus 事件**:
  - [ ] 注册 `module:action` 调试事件
- **GM 指令**:
  - [ ] 注册 `GM.xxx()` 调试指令
- **实现方案**: （AI 领取后填写）
- **完成摘要**: （AI 完成后填写）
```

## 字段说明

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 任务编号 | 是 | Sprint 内部递增，格式 `T1` / `T2` / ... |
| 状态 | 是 | 🔵/🟡/🟢/🔴 |
| 类型 | 是 | 功能/修复/重构/美术/设计/优化 |
| 点数 | 是 | 工作量预估（整数） |
| 优先级 | 是 | P0/P1/P2 |
| 验收标准 | 是 | Checklist 格式，AI 自检用 |
| DebugBus 事件 | 功能/修复类必填 | 需要注册哪些调试事件 |
| GM 指令 | 功能类必填 | 需要注册哪些 GM 指令 |
| 实现方案 | AI 填写 | 领取任务后补充技术方案 |
| 完成摘要 | AI 填写 | 完成后记录做了什么、注意事项 |
